# 프로젝트 개요: Lotto645 시뮬레이션

이 문서는 `Lotto645` 프로젝트의 개요, 기술 스택, 핵심 기능 및 구조를 설명합니다.

## 1. 프로젝트 설명

**Lotto645**는 웹 기반의 로또 6/45 시뮬레이션 애플리케이션입니다. 사용자는 로또 티켓을 발급받을 수 있으며, 관리자는 실시간으로 추첨을 진행하고 결과를 확인할 수 있습니다. 모든 과정은 Firebase 실시간 데이터베이스와 연동되어 여러 사용자에게 동기화됩니다.

## 2. 핵심 기능

- **티켓 발행 페이지 (`/issue`):**
  - 1부터 45까지의 숫자 중 6개를 수동으로 선택할 수 있습니다.
  - `자동 선택` 기능을 통해 부족한 개수만큼 숫자를 랜덤으로 채울 수 있습니다.
  - 발행자 이름을 입력하고 티켓을 발행하면 정보가 Firebase에 저장됩니다.

- **추첨 대시보드 페이지 (`/draw`):**
  - 발행된 모든 티켓 목록을 실시간으로 확인할 수 있습니다.
  - `추첨 시작` 버튼을 누르면 5초 간격으로 룰렛 애니메이션과 함께 6개의 고유한 당첨 번호가 추첨됩니다.
  - 추첨된 번호는 티켓 목록에 실시간으로 하이라이트 처리됩니다.
  - `초기화` 버튼을 통해 현재 세션의 모든 티켓과 추첨 데이터를 삭제할 수 있습니다.

## 3. 기술 스택

- **프론트엔드:** React (TypeScript), Vite
- **데이터베이스:** Firebase 실시간 데이터베이스
- **테스팅:**
  - 단위 테스트: Vitest
  - End-to-End (E2E) 테스트: Playwright

## 4. 데이터 모델

주요 데이터는 Firebase 실시간 데이터베이스의 단일 세션(`sessions/current`) 하위에 저장됩니다.

- **Ticket (`/sessions/current/tickets/{ticketId}`):**
  - `name`: 발행자 이름
  - `numbers`: 선택된 6개의 숫자 배열
  - `createdAt`: 생성 타임스탬프

- **DrawResult (`/sessions/current/draw`):**
  - `numbers`: 추첨된 숫자 배열
  - `status`: 추첨 상태 (`idle`, `in_progress`, `completed`)
  - `startedAt`, `completedAt`: 추첨 시작 및 완료 타임스탬프

## 5. 프로젝트 구조

- `frontend/`: React 애플리케이션 소스 코드
- `tests/e2e/`: Playwright E2E 테스트 코드
- `specs/`: 기능 명세, 데이터 모델, 구현 계획 등 상세 문서
- `docs/`: Firebase 연동 가이드 등 추가 문서
